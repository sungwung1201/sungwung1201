### 1️⃣ SNFC - IsaacSim 기반 다중 AMR 물류 자동화 시스템
> **AMR 5대, SH5 로봇팔 3대, Control Tower, PostgreSQL/Redis, FastAPI Dashboard를 연결한 통합 물류 자동화 시뮬레이션 프로젝트**

* **Role:** 팀장 / 프로젝트 기획 / 전체 시나리오 설계 / AMR Fleet / 문서화
* **Tech Stack:** `Python`, `ROS2 Humble`, `IsaacSim`, `CycloneDDS`, `PostgreSQL`, `Redis`, `FastAPI`, `Docker`
* **Main Contribution:**
  * **프로젝트 기획 및 시나리오 설계:** 입고, 작업대 적재, AMR 이송, SH5 Pick & Place, 출고, Dashboard 모니터링까지 이어지는 전체 물류 자동화 흐름을 설계.
  * **AMR Fleet 구조 설계:** IsaacSim 환경에서 `AMR_01` ~ `AMR_05`가 작업대를 픽업하고 목표 위치로 운반한 뒤 복귀하는 다중 AMR 운영 구조 정리.
  * **Time A* / Reservation Table:** 단순 최단 경로가 아니라 시간축을 포함한 경로계획을 사용하여 미래 cell 점유와 edge swap 충돌을 고려.
  * **Global Arbiter:** 각 AMR의 다음 이동 후보를 전역적으로 검사하여 같은 cell 점유, edge swap, 작업대 footprint 충돌을 방지.
  * **Cost-aware Reroute:** 막히면 무조건 기다리는 방식이 아니라, `wait_cost`와 `detour_cost`를 비교하여 기다릴지 우회할지 판단하는 구조 정리.
  * **Bridge Admission Guard:** 같은 작업대, 같은 target, 같은 AMR에 대한 중복 명령을 Bridge 단계에서 차단하여 물리적으로 불가능한 명령이 Controller로 들어가지 않도록 개선.
  * **통신 및 통합 디버깅:** CycloneDDS Wi-Fi 설정, ROS2 Action Server discovery, Bridge Queue, IsaacSim Controller 연동 문제를 분석하고 정리.
  * **문서화:** README, DEBUGGING, GitHub 업로드 문서, 발표 PPT, 발표 대본, 시스템 아키텍처, 플로우차트 정리를 주도.

```text
Control Tower 작업 생성
→ Redis ZSET 우선순위 스케줄링
→ ROS2 Action / Service 명령 송신
→ AMR Bridge가 명령을 JSON Queue로 변환
→ IsaacSim AMR Controller가 Time A* / Global Arbiter 기반 작업대 이송
→ SH5 3대가 Pick & Place 수행
→ PostgreSQL / Redis 상태 갱신
→ Dashboard에서 실시간 모니터링
```

### 2️⃣ TurtleBot4 박물관 보안 순찰 및 침입자 추적 시스템
> **두 대의 TurtleBot4 AMR이 박물관을 번갈아 순찰하고, 도난 상황 발생 시 침입자 위치를 추적하는 ROS2 기반 보안 순찰 시스템**

* **Role:** AMR 이동 알고리즘 / Nav2 순찰·추적 제어
* **Tech Stack:** `Python`, `ROS2 Humble`, `TurtleBot4`, `Nav2`, `AMCL`, `YOLO`, `RGB-D`, `TF2`
* **Main Contribution:**
  * **Waypoint Patrol:** TurtleBot4가 사전에 정의된 waypoint를 순차적으로 이동하도록 Nav2 `NavigateToPose` 기반 순찰 루프 구현.
  * **AMCL Initial Pose:** `/robot2/initialpose` 자동 발행으로 RViz 2D Pose Estimate 수동 설정 의존도 감소.
  * **Arrive Trigger:** 특정 유물 위치 도착 시 `/robot2/arrive_position`을 발행하여 AMR 이동과 비전 노드의 유물 확인 시점을 동기화.
  * **Multi-Robot Patrol Handoff:** robot2 순찰 완료 후 `/robot8/start_patrol_signal`을 발행하여 두 대의 TurtleBot이 번갈아 순찰하도록 구성.
  * **Chase Mode 전환:** 도둑 좌표 수신 시 기존 Nav2 goal을 `cancelTask()`로 취소하고 chase mode로 전환.
  * **Dock / Undock 처리:** dock 상태에서 상황이 발생하면 자동으로 undock 후 추적 시작.
  * **Tracking Goal Update:** `thief_position`과 `/robot2/amcl_pose`를 이용해 침입자와 일정 거리 떨어진 추적 goal을 계산하고 반복 갱신.

```text
/start_patrol_signal 수신
→ AMCL initial pose 설정
→ Nav2 waypoint 순찰
→ arrive_position 발행
→ thief_position 수신
→ 기존 goal 취소
→ chase mode 전환
→ 침입자 기준 추적 goal 반복 갱신
```

### 3️⃣ F1TENTH ROS2 Autonomous Driving
> **ROS2 기반 F1TENTH Gym 환경에서 LiDAR와 Odometry 데이터를 직접 처리하여 자율주행 알고리즘을 구현한 프로젝트**

* **Role:** 자율주행 알고리즘 구현 / 경로계획 / 경로추종 / 디버깅
* **Tech Stack:** `Python`, `ROS2 Humble`, `F1TENTH Gym`, `LiDAR`, `Odometry`, `RRT*`, `Pure Pursuit`, `AckermannDriveStamped`
* **Main Contribution:**
  * **Safety Node:** LiDAR와 Odometry 기반 TTC(Time-To-Collision)를 계산하여 충돌 위험 시 `/drive`로 긴급 정지 명령 발행.
  * **Wall Following:** LiDAR 특정 각도 거리값을 이용해 벽과의 거리 오차를 계산하고 PID 기반 조향 제어 수행.
  * **Follow-the-Gap:** 전방 LiDAR 영역에서 Safety Bubble을 적용하고 가장 넓은 free space를 찾아 장애물 회피.
  * **Pure Pursuit:** waypoint CSV와 Odometry 기반 현재 pose를 이용해 lookahead point를 선택하고 Ackermann steering angle 계산.
  * **RRT / RRT*:** Occupancy Grid 기반 충돌 검사, parent selection, rewiring을 적용해 경로 품질 개선.
  * **Obstacle Inflation:** 차량 중심점 기준이 아닌 차량 body 기준 안전 여유를 고려해 장애물 주변 영역 확장.
  * **Nav2 / RL 미사용 구분:** Nav2 전체 스택이나 강화학습 policy가 아니라 직접 구현한 고전 자율주행 알고리즘으로 구성.

```text
/scan LiDAR 수신
→ 장애물 거리 및 TTC 계산
→ Safety Bubble / Gap 탐색
→ Occupancy Grid 충돌 검사
→ RRT* 경로계획
→ Pure Pursuit 경로추종
→ /drive Ackermann 제어
```

### 5️⃣ ROBOCASE - Doosan M0609 폰케이스 드로잉 자동화
> **사용자가 웹 앱에서 만든 도안을 서버에 저장하고, Doosan M0609 협동로봇이 실제 폰케이스 위에 자동으로 그리는 프로젝트**

* **Role:** 팀장 / 프로젝트 기획 / 전체 시나리오 설계 / 로봇 공정 설계 / 발표 구성
* **Tech Stack:** `Python`, `ROS2 Humble`, `Doosan M0609`, `Flask`, `SQLite`, `HTML5 Canvas`, `OpenCV`, `MoveLine`
* **Main Contribution:**
  * **프로젝트 기획:** 사용자가 직접 만든 도안이 실제 로봇 드로잉으로 이어지는 Web-to-Robot 제작 공정 시나리오 설계.
  * **Phone → Server → Robot 구조:** Canvas 이미지와 Stroke JSON을 서버에 저장하고, 로봇 제어 노드가 주문을 감지해 작업을 수행하는 흐름 정리.
  * **로봇 공정 설계:** 케이스 픽업, 작업대 세팅, 펜 픽업, 드로잉, 펜 반납, 완성품 배출, HOME 복귀 단계 설계.
  * **좌표 변환 방향 설계:** Canvas pixel 좌표를 실제 로봇 mm 좌표로 변환하는 구조 정리.
  * **Path 생성 알고리즘 디버깅:** Stroke JSON 우선 방식과 OpenCV fallback 방식을 구분하고, HSV mask, Boundary Centerline, Catmull-Rom, Resampling, Path Connect를 검토.
  * **MoveLine 제어 구조:** MoveSplineTask보다 실제 로봇에서 안정적인 MoveLine 기반 드로잉 제어로 단일화.
  * **예외처리:** 주문 취소, MoveStop, 외부 충격 정지, HOME 복귀 흐름을 서버 상태와 연결.

```text
Phone App에서 도안 생성
→ Flask Server가 이미지와 Stroke JSON 저장
→ RobotDrawerNode가 주문 감지
→ Stroke JSON 또는 OpenCV로 path 생성
→ Pixel 좌표를 Robot mm 좌표로 변환
→ M0609 MoveLine 드로잉
→ 완료 / 취소 / 충격 정지 상태 반영
```
